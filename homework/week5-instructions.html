Week 5 Homework

Your hot new rideshare startup, Noober, is a huge success. The ride requests are pouring in. But your drivers are complaining that it's taking too long to look through all the rides. You need a quick solution and after a few meetings, the team has determined the quickest feature to implement will be buttons at the top of the rides list to filter the list by level of service.

Getting Started

Begin by heading over to the starter project – select "Use this template", then create a repository in your own GitHub account called hw5. Clone this project and open it in VSCode.

Open index.html in Chrome. You'll notice your application is very similar to where you left off last week, i.e. the full list of all active ride requests in the database. Your designers have added the level of service button filters at the top, but they don't do anything yet when clicked.

The Goal

On page load, no ride data will appear. Instead, you should see an "All Rides" button, a button for each level of service. When a user (i.e. driver) clicks on each button, the list of rides should appear just as it did last week, except filtered:

  • "All Rides" should display all rides
  • Each level of service button should filter the rides by the given level of service

The ride data can be retrieved from our "API" endpoint found at the URL https://kiei451.com/api/rides.json. The code to render the html of all the rides has already been provided in a renderRides() function. Your job is to add an event listener for each button, and when the event occurs, fetch the data, filter out any undesired rides based on the selected filter, and then call the renderRides() function, passing an array of the remaining rides as the expected argument.

Rubric

The rubric below is intended as a guide to a solution. But there are many possible solutions. What ultimately matters is that your application successfully displays the expected/filtered ride data when each button is clicked.

  1. (1 point) Add an event listener to the "All Rides" button, so that when it is clicked, there is some output in the console to indicate which button was clicked.
  2. (1 point) When the event occurs:
    1. request the ride data from our "API"
    2. pass the array of rides to the provided renderRides() function to display all the rides (same as they appeared at the end of last week's assignment)
  3. (2 points) Add an event listener to the "Noober Purple" button, so that when it is clicked, there is some output in the console to indicate which button was clicked.
  4. (4 points) When the event occurs:
    1. request the ride data from our "API"
    2. create a new empty array
    3. loop through the rides and for each ride, use the provided levelOfService() function to determine the service level, and use newArray.push(ride) to add "Noober Purple" rides into the new array
    4. lastly, pass the new array of filtered rides to the renderRides() function
  5. (2 points) Repeat steps 3 & 4 for the other 3 level of service button filters.

Optional Challenges

These are not graded, but if you want to stretch your skills...

Challenge 1: It would be a nicer UX (User Experience) if when a button is selected, it was clear to the user. In the event listener, remove/add a Tailwind class for the clicked button so that its background color appears to be active. However, make sure to first reset all the buttons to their original background color (or else it will be confusing which button was selected).

Challenge 2: Instead of having the code for 5 individual event listeners (1 for each button), can you refactor the code to use a loop and only write code for 2 event listeners? Or even just 1?

Guidelines for Submission

All your work should take place in noober.js; i.e. that should be the only file that's modified from the original project template. Your finished product should live in your GitHub account at https://github.com/<your GitHub username>/hw5, and this URL should be submitted as the "website URL" for this assignment in Canvas.
